1. What HelixSeed Does

HelixSeed searches Minecraft world seeds for ones that contain the structures and biomes you want. It runs a fast GPU prefilter to throw away seeds that obviously do not match, then a strict CPU verification pass on the survivors so the seeds it reports really do contain what you asked for.

You describe what you want using either the Builder tab (point-and-click constraints) or the Query tab (a small scripting language called HelixScript). Both produce the same compiled query under the hood.

2. Quick Start (5 Minutes)

  1. Open HelixSeed. The first thing you see is the Builder tab.
  2. In the Presets panel on the left, click Starter. This drops in a "village + nearby ruined portal" search.
  3. Set Mode (top dropdown) to strict. Strict means HelixSeed only reports seeds that fully match.
  4. Set Count to something small, like 1000000, so the first run finishes quickly.
  5. Press the Run button.
  6. Switch to the Results tab. Found seeds appear there as you scan; click any seed to copy it to the clipboard.
Nothing happening? Open the Telemetry tab — if seeds-per-second is 0, the scanner failed to launch. Check that native\scanner_native.exe exists in the install folder and that your GPU drivers are up to date.

3. App Tour

Sidebar (left)

The sidebar holds Run controls (Run / Pause / Stop), Scan settings (mode, count, batch, output, version), and the Presets panel. Anything you can change here also has a HelixScript equivalent.

Builder tab

Add structure constraints (e.g. "village within 96 of origin") and biome filters (e.g. "must be cherry_grove near origin"). Each row has a mode dropdown:

  • strict — exactly verify the structure exists (slower, no false positives).
  • placement — quick check that the structure could generate near there (faster, may report false positives).
  • mixed — alias for strict at the per-constraint level. The global Mode dropdown is where mixed actually splits work.

Query tab

This is the HelixScript editor. Builder rows compile to HelixScript automatically; you can edit the script directly and tick "Run the script editor directly" to bypass the builder. Use the block menu on the right to insert find, biome, for, scan, and other building blocks. Open the HelixScript reference for the full language details.

Results tab

Found seeds stream in here as the scanner discovers them. The filter bar lets you split placement survivors from strict-verified seeds when you run mixed mode. Click a seed to copy it.

Telemetry tab

Live charts of seeds-per-second, queue depth, GPU survivors, strict throughput, and Java-validation rejects. Use it to compare performance settings — change one knob, watch the rate move.

Documentation tab

Opens this guide in a separate window so you can keep it visible while you work. From here you can jump to the HelixScript reference for the scripting language details.

4. Presets

Presets are one-click starting points. Two kinds:

Search presets (replace your constraints)

Starter

Village near origin plus a ruined portal close to that village. A reliable smoke test.

Trial

Trial chambers within 256 of origin. Uses placement mode so it screams through seeds.

Ancient

Ancient city within 512 of origin, plus a cherry grove / meadow biome filter.

Performance presets (replace your engine settings)

Max Performance

Pushes every engine knob: max-throughput execution, multi-stage A, async GPU pipeline, lightspeed surrogate, max batch. Best for long unattended scans on a strong machine.

Smart Mode

Switches the global scan mode to mixed and applies Max Performance on top. The recommended "just go fast" setting — placement filters out junk seeds, strict only verifies the survivors.

Performance presets only touch engine settings, not your constraints. Combine them with a search preset (or your own constraints) to scan immediately.

5. Tips

  • Start small. Set count to a million for the first run so you know the scanner is healthy before you commit to a long search.
  • Use Smart Mode when you do not care about the exact engine settings — it is the right answer 90% of the time.
  • The Telemetry tab is your friend. If your CPU usage is low, try changing execution mode or strict workers. If the GPU survivor rate looks suspicious, change strict surrogate or move to mixed mode.
  • Save your script. The Query tab is just text — copy it into a file, you can paste it back any time.
  • Pause (P) to inspect telemetry without losing scan state. Stop halts entirely.
  • Switch versions using the MC version field if you are searching for an older Minecraft version.
